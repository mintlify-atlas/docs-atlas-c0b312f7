## Get up and running

<Steps>
  <Step title="Import Defaults">
    Add the import at the top of any file where you want to read or write stored values:

    ```swift
    import Defaults
    ```
  </Step>
  <Step title="Define your keys">
    Extend `Defaults.Keys` with static `Key<Type>` declarations. Each key has a name, a type, and a default value.

    ```swift
    extension Defaults.Keys {
        static let quality = Key<Double>("quality", default: 0.8)
        static let isUnicornMode = Key<Bool>("isUnicornMode", default: false)
    }
    ```

    <Tip>
      Define all your keys in a single file (for example, `DefaultsKeys.swift`). This makes them easy to discover and prevents accidental duplication.
    </Tip>

    <Note>
      Key names must be ASCII, must not start with `@`, and cannot contain a dot (`.`). Violating these rules produces a runtime warning.
    </Note>
  </Step>
  <Step title="Read and write values">
    Use the `Defaults` subscript to get or set values anywhere in your app — no `UserDefaults` boilerplate required.

    ```swift
    Defaults[.quality]        // => 0.8 (returns the default on first run)
    Defaults[.quality] = 0.5  // => 0.5
    Defaults[.quality] += 0.1 // => 0.6

    Defaults[.quality] = "🦄"
    // [Cannot assign value of type 'String' to type 'Double']
    ```
  </Step>
  <Step title="Use in SwiftUI">
    The `@Default` property wrapper works like `@State` — the view re-renders automatically whenever the stored value changes.

    ```swift
    struct ContentView: View {
        @Default(.quality) var quality

        var body: some View {
            Slider(value: $quality, in: 0...1)
        }
    }
    ```

    You can also reset a key to its default value from a button:

    ```swift
    struct ContentView: View {
        @Default(.quality) var quality

        var body: some View {
            VStack {
                Slider(value: $quality, in: 0...1)
                Button("Reset") {
                    _quality.reset()
                }
            }
        }
    }
    ```
  </Step>
  <Step title="Observe changes">
    Use `Defaults.updates(_:)` to receive a value every time a key changes. It returns an `AsyncStream`, so you iterate it with `for await` inside a `Task`.

    ```swift
    extension Defaults.Keys {
        static let isUnicornMode = Key<Bool>("isUnicornMode", default: false)
    }

    Task {
        for await value in Defaults.updates(.isUnicornMode) {
            print("Value:", value)
        }
    }
    ```

    By default, `updates` emits the current value immediately when the stream starts (`initial: true`). Pass `initial: false` to receive only future changes.
  </Step>
</Steps>

## Storing an enum

Any `Codable` enum (or struct) works out of the box. Conform to `Defaults.Serializable` and Defaults picks up the `CodableBridge` automatically:

```swift
enum Theme: String, Codable, Defaults.Serializable {
    case light
    case dark
    case system
}

extension Defaults.Keys {
    static let theme = Key<Theme>("theme", default: .system)
}

Defaults[.theme]           // => .system
Defaults[.theme] = .dark   // => .dark
```

## Resetting to the default value

Call `Defaults.reset` with one or more keys to remove the stored value and revert to the declared default:

```swift
Defaults[.quality] = 0.1
Defaults[.quality]           // => 0.1

Defaults.reset(.quality)

Defaults[.quality]           // => 0.8 (back to the default)
```

You can reset multiple keys in a single call:

```swift
Defaults.reset(.quality, .isUnicornMode, .theme)
```
