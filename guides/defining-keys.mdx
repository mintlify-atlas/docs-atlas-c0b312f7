---
title: "Defining Keys"
---

Keys are the foundation of the Defaults library. Each key has a name, a type, a default value, and an optional `UserDefaults` suite. You declare them once and use them everywhere.

## Basic key declaration

Extend `Defaults.Keys` and add `static` properties of type `Key<Value>`:

```swift
import Defaults

extension Defaults.Keys {
    static let quality = Key<Double>("quality", default: 0.8)
    //            ^            ^         ^                ^
    //           Key          Type   UserDefaults name   Default value
}
```

Once declared, read and write values using the subscript syntax:

```swift
Defaults[.quality]        // => 0.8
Defaults[.quality] = 0.5  // => 0.5
Defaults[.quality] += 0.1 // => 0.6

// Type mismatch is caught at compile time:
Defaults[.quality] = "🦄"
// [Cannot assign value of type 'String' to type 'Double']
```

## Key name rules

The name string passed to `Key` must follow these rules:

- Must be ASCII characters only
- Must **not** start with `@`
- Must **not** contain a dot (`.`)

Violating any of these rules triggers a runtime warning.

## Default values

### Static default values

Pass a literal or constant as the `default:` argument:

```swift
extension Defaults.Keys {
    static let isUnicornMode = Key<Bool>("isUnicornMode", default: false)
    static let username = Key<String>("username", default: "Guest")
    static let maxRetries = Key<Int>("maxRetries", default: 3)
}
```

The default value is registered with `UserDefaults` via `register(defaults:)`, making it available to other frameworks (e.g., KVO bindings) that read the store directly.

### Dynamic default values

When the default value cannot be determined statically — for example, because it depends on system state that may change — use the closure-based initializer:

```swift
extension Defaults.Keys {
    static let camera = Key<AVCaptureDevice?>("camera") {
        .default(for: .video)
    }
}
```

> **Note:** The dynamic initializer does **not** register the default value with `UserDefaults`. This only matters if you rely on legacy KVO bindings that read the store directly.

## Optional value keys

For optional types, omit the `default:` argument entirely. The library provides a convenience initializer that defaults to `nil`:

```swift
extension Defaults.Keys {
    static let selectedTag = Key<String?>("selectedTag")
    static let lastSyncDate = Key<Date?>("lastSyncDate")
}
```

These keys start as `nil` when no value has been stored yet.

## Custom UserDefaults suite

Each key can target a specific `UserDefaults` suite, for example an App Group suite shared between a host app and an extension:

```swift
extension Defaults.Keys {
    static let sharedToken = Key<String>(
        "sharedToken",
        default: "",
        suite: UserDefaults(suiteName: "group.com.example.app")!
    )
}
```

When no `suite` is provided, `.standard` is used.

## Checking whether a value is the default

For keys whose `Value` type conforms to `Equatable`, the `isDefaultValue` property tells you whether the current stored value equals the default:

```swift
extension Defaults.Keys {
    static let quality = Key<Double>("quality", default: 0.8)
}

Defaults[.quality]          // => 0.8
Defaults.Keys.quality.isDefaultValue // => true

Defaults[.quality] = 0.5
Defaults.Keys.quality.isDefaultValue // => false
```

## Type-erased base class: `_AnyKey` / `Keys`

`Defaults.Key<Value>` is generic, so you cannot store keys of different types in the same Swift collection. The type-erased base class `Defaults._AnyKey` (aliased as `Defaults.Keys`) solves this:

```swift
let keysToReset: [Defaults.Keys] = [
    Defaults.Keys.quality,
    Defaults.Keys.isUnicornMode,
]

Defaults.reset(keysToReset)
```

`_AnyKey` exposes `name: String` and `suite: UserDefaults`, and provides a `reset()` method that removes the stored value, restoring the key to its default.

## Organizing keys

A common pattern is to keep all keys in a single file so they are easy to discover and audit:

```swift
// AppDefaults.swift
import Defaults

extension Defaults.Keys {
    // Appearance
    static let accentColor = Key<String>("accentColor", default: "blue")
    static let fontSize = Key<Double>("fontSize", default: 14)

    // Behavior
    static let autoSave = Key<Bool>("autoSave", default: true)
    static let syncInterval = Key<Int>("syncInterval", default: 60)
}
```

Alternatively, declare keys in extensions per feature alongside the feature code:

```swift
// FeatureA/Settings.swift
extension Defaults.Keys {
    static let featureAEnabled = Key<Bool>("featureAEnabled", default: false)
}
```
